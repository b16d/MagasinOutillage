# ADR-001 — Java 25

## Statut
Accepté

## Contexte
Le projet nécessite un langage robuste pour le développement back-end.
Java 25 est la dernière version LTS (Long-Term Support) disponible.

## Décision
Utiliser Java 25 comme langage de développement.

## Conséquences
- Accès aux dernières fonctionnalités du langage : records, sealed classes, pattern matching, virtual threads
- Support LTS garanti sur le long terme
- Large écosystème de bibliothèques et frameworks compatibles
- Nécessite un JDK 25 installé sur les environnements de développement et de déploiement
